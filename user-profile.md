# User Profile

## Overview
This document describes the user profile feature for the test1 repository.

## Profile Structure

A user profile contains the following information:

- **Name**: User's full name
- **Username**: Unique identifier
- **Email**: Contact email address
- **Bio**: Short description about the user

## Example Profile

```markdown
**Name**: John Doe  
**Username**: johndoe  
**Email**: john.doe@example.com  
**Bio**: Software developer and open source enthusiast
```

## Usage

User profiles can be created by adding a new file in the `profiles/` directory with the naming convention `username.md` (hyphens are allowed in usernames, e.g., `example-user.md`).

## Features

- Simple markdown-based profile system
- Easy to read and edit
- Version controlled through git
- Minimal dependencies
